# SOP: Blog Drafting & Publishing Workflow

This Standard Operating Procedure ensures that every blog post is SEO-optimized, technically sound, and correctly deployed to the live Pak Homies Industry website.

## 1. Research & Drafting
- **Keywords**: Use the `analysis_results.md` to select a high-intent, low-competition long-tail keyword.
- **Content**: Draft the content in a markdown file in the `blog_drafts/` directory.
- **SEO Check**: Ensure the draft includes:
    - Target H1 Title
    - Meta Description (150-160 chars)
    - Strategic internal links to Service or Product pages.

## 2. Implementation (Code)
- **Component**: Create a new `.tsx` file in `client/src/pages/blog/` (e.g., `BlogMyNewTopic.tsx`). Use the `BlogHeavyweightHoodie.tsx` as a template for premium styling and animations.
- **Routing**: Import and add the new route in `client/src/App.tsx`.
- **Index**: Add the post metadata to the `posts` array in `client/src/pages/Blog.tsx` to make it appear on the main blog feed.

## 3. CRITICAL: Technical Deployment (The "Push" Step)
The live site on Hostinger requires a manual compilation of assets. **Skipping this step will result in "Article Not Found" errors.**

1.  **Build Locally**: Run the following command in the project root:
    ```bash
    npm run build:local
    ```
    *This generates the production-ready files in the `dist/` folder.*

2.  **Commit & Push**: Add ALL changed files (including `src` and `dist`) to git:
    ```bash
    git add .
    git commit -m "feat(blog): publish [Blog Title]"
    git push origin main
    ```

## 4. Post-Publishing Verification
- Navigate to `pakhomiesind.com/blog` to ensure the post is visible.
- Click the new post to verify the URL and content rendering.

# 🔍 Will Google find this blog automatically?

**Yes, but with some conditions:**

1.  **Crawling**: Google's bots (Googlebot) periodically crawl your site. When they visit your [Blog Index](https://pakhomiesind.com/blog) and see the new link, they will follow it and index the page.
2.  **Sitemap**: Your site likely has a `sitemap.xml`. While I updated the code, some sitemaps are static and may need manual updating, or they might be auto-generated by the build. 
3.  **Speed Up Indexing**: To get ranked faster (within minutes/hours instead of days):
    - Go to **Google Search Console**.
    - In the **URL Inspection** tool, paste your new blog URL.
    - Click **"Request Indexing"**.

> [!TIP]
> Sharing the new blog post on LinkedIn, WhatsApp, or other social channels also creates "social signals" that help Google find and trust the new content faster.
